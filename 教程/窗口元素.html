窗口元素绘图

介绍完普通绘图后，如果那这些进行绘图就会发现有很多不方便的地方，绘制一个图形会使用大量的代码，当图形稍微多一点就会很乱，不够整洁优雅，而且功能残缺。
使用元素法绘图可以把每个图形单独抽象为一个元素，方便管理。

内容

  • 简单绘制示例
  • 图片
  • 文本
  • 折线
  • 矩形
  • 椭圆

利用[d2dPicture]绘制一张图片

首先在主函数中定义一个[AutoInit]变量来初始化位图设备

AutoInit autoInit;

[AutoInit]

再定义一个结构体或者类存储窗口元素

//窗口元素的命名空间
using namespace Game::WindElements;
struct WindowElements
{
	//背景
	d2dPicture background;
	//跟随鼠标移动的正五边形
	d2dGeometry regularPentagon;
};

[d2dGeometry]

然后在构造函数中定义属性，要先设置颜色，否则不会显示

WindowElements(MainWind_D2D* window)
{
    //从文件加载图片作为背景
    background.LoadPictureFromFile(L"背景.jpg", window);
    //设置五边形的颜色
    regularPentagon.SetColor(D2D1::ColorF(0.7, 0, 0.7), window->GetD2DTargetP());
    //设置为填充模式绘制
    regularPentagon.Fill(true);
    //计算五边形的点
    std::vector<D2D1_POINT_2F> points(5);
    float angle = 3.14159 * 2 / 5;
    float size = 100;
    for (int i = 0; i < 5; ++i)
    {
        points[i] = D2D1::Point2F((size * std::sin(angle * i)), (-size * std::cos(angle * i)));
    }
    //将绘制路径放入
    regularPentagon.SetPath(window,points);
}

填充模式后面可以加填充颜色，没有就默认纯色

把窗口用户数据替换为此结构体的指针
再分别修改绘图回调与鼠标回调

void PaintWindow(MainWind_D2D* window)
{
	//绘制一个正五边形
	WindowElements* windowElements = (WindowElements*)window->GetUserData();
	windowElements->background.Draw(window);
	windowElements->regularPentagon.Draw(window);
}

void MouseMove(MainWind* window,int x,int y,int,MouseMessageType type,KeyMode mode)
{
	if (type == MT_MOVE)
	{
		WindowElements* windowElements = (WindowElements*)window->GetUserData();
		windowElements->regularPentagon.SetPosition(x, y);
	}
}

这样就绘制出了一个带有图片背景的跟随鼠标移动图案的窗口

如果你实际上手操作后可能会发现背景图片对不齐
由于加载位图后默认会把绘制区域设置为图片的大小，要让图片的大小与窗口对应就需要设置窗口大小回调，让图片的显示大小跟随窗口做出调整

首先更具d2d窗口窗口大小调整回调的函数定义要求定义函数

void WindowSize(MainWind_D2D* window,int w,int h)
{
	WindowElements* windowElements = (WindowElements*)window->GetUserData();
	windowElements->background.SetShowWide(w, h);
}

然后一样的，再与窗口绑定

Window.SetWindSizeCallback(WindowSize);

这样理论上就可以就能正确的显示背景了
但是我们发现并没有正确显示，只有调整窗口大小后才对。这是由于在创建窗口时没有触发窗口大小调整回调，所有我们在初始化时就设置图片的大小就行了

我们在加载图片函数后面加上这个函数

auto windowSize = window->GetWindSize();
background.SetShowWide(windowSize.cx, windowSize.cy);

这样就正常显示了

其他元素使用示例

d2dText

类似的，先设置颜色然后设置显示的信息

text.SetColor(D2D1::ColorF(0.1, 0.1, 0), window->GetD2DTargetP());
text.SetShowText(L"d2dText使用示例");

然后记得设置显示位置，否则默认为0，0，这里使用的是上面跟随鼠标移动的代码，就没有额外设置。

然后记得在绘图回调中添加绘制函数

windowElements->text.Draw(window);

d2dFoldLine

折线的绘制
这里以绘制一个sin图象为例

一样的，先设置颜色

Lines.SetColor(D2D1::ColorF(1, 0, 0), window->GetD2DTargetP());

定义连续的点数组

std::vector<D2D1_POINT_2F> points(30);
const float step = (float)(windowSize.cx - 100) / points.size();
const float start = 50;
const float amplitude = 100;
const float cycle = 3;
for (int i = 0; i < points.size(); ++i)
{
    points[i] = D2D1::Point2F(
        start + step * i, 
        amplitude * std::sin(i * 3.14159 * 2 * cycle / points.size()));
}
//设置绘制位置
Lines.SetPosition(0, windowSize.cy * 0.5);

然后将点赋值给折线元素

Lines.SetFoldLine(points);

这样就能正确的绘制一个类sin曲线了

更多的样式的绘制
可以使用与简单绘图类似的函数SetPen,
不过这里的SetPen需要将窗口作为第一个参数

Lines.SetPen(window,D2D1::ColorF(1,0,1),PenStyle::DashDotDot);

也可以使用SetPenStyle来自定义曲线，同样不同的是，第一个参数需要为窗口或d2d工厂

比如这里绘制先点后线的渐变曲线

Lines.SetPenStyle(
    window,
    D2D1::StrokeStyleProperties(
        D2D1_CAP_STYLE_FLAT,
        D2D1_CAP_STYLE_FLAT,
        D2D1_CAP_STYLE_ROUND,
        D2D1_LINE_JOIN_MITER,
        10.0f,
        D2D1_DASH_STYLE_CUSTOM,
        0.0f),
	{1,6,6,7,9,8,12,27,18,30});

d2dRectangle

矩形元素，支持绘制圆角矩形以及旋转绘制
都是一样的步骤，设置颜色，设置大小，设置位置(由于位置是跟随鼠标的，没有单独设置)

rectangle.SetColor(D2D1::ColorF(0.7, 0.7, 0), window->GetD2DTargetP());
rectangle.SetShowWide(200, 100);

然后尝试设置为填充模式绘制并设置圆角

//启用圆角
rectangle.Round(true);
//设置x方向与y方向的圆角弧度
rectangle.SetRound(30, 15);

现在就是一个填充了颜色的并且带有圆角的矩形了

你可能会发现简单绘制能设置填充色与边框色，这个似乎没有说，这是因为简单绘制是由边框与填充两个矩形构成的，所以如果需要绘制边框，可以使用两个窗口元素

矩形元素还支持旋转

//设置旋转角度与旋转中心
rectangle.SetRotate(90, D2D1::Point2F(0.5,0.5));
//旋转中心以矩形大小为基准的百分比坐标
//比如（0，0）表示矩形左上角，（0.5，0）表示矩形的顶部边形的中点

d2dElliptic

仍然是相同的步骤，先设置颜色，再设置椭圆信息

//设置颜色
elliptic.SetColor(D2D1::ColorF(0.7, 0.1, 0), window->GetD2DTargetP());
//设置x轴与y轴
elliptic.SetRadius(200, 100);
//设置中点位置
elliptic.SetEllipticalPosition(500, 300);

依旧可以设置填充模式，这里不放图片了

d2dPicture

图片元素，这个在控件与场景中被作为了资源使用

图片的绘制首先需要加载图片资源，可以从内存加载或从文件加载

从文件加载
[LoadPictureFromFile]
从内存加载位图数据，数据由像素点组成
[LoadBitmapFromMemory]
从内存加载png，jpg等
[LoadImageFromMemory]

设置显示的切片
[SetCrop]
比如裁取一半

picture.LoadPictureFromFile(L"图片.png", window);
auto pictureSize = picture.GetImageSize();
picture.SetCrop(D2D1::RectF(0, 0, pictureSize.width * 0.5, pictureSize.height * 0.5));

图片的显示大小是使用[SetShowWide]设置的
