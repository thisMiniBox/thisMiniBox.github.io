窗口控件的使用

介绍各种窗口控件的使用示例

内容

  • 图片背景按钮
  • 纯色背景按钮
  • 函数显示
  • 编辑框控件
  • 动画控件

按钮控件

用于检测用户点击并作出反馈的控件

图片控件使用Init函数进行初始化

图片按钮

d2dImageButton
以图片为背景的按钮控件

初始化图片按钮

先自定义一个按键回调
因为在类中，所以使用static
按键回调所需的定义

static void ButtonCallback(MainWind* window,long long)
{
    std::cout << "点击了按钮" << std::endl;
}

初始化

imageButton.Init(
    300, 200,	//显示的位置-x-y
    100, 50,	//显示的大小-宽度-高度
    L"按钮", D2D1::ColorF(1,0,0),//显示的文本-文本-颜色
    ButtonCallback,//点击回调
    window,		//窗口
    L"按钮.png",//按钮的背景图片文件地址
    0			//用户数据
);

然后记得在绘图回调中调用绘制函数就可以正常显示了
但还不能正常使用，需要与使用[Bind]函数与窗口绑定
绑定后回调函数才会正常调用
绑定多个窗口会自动调用[Unbind]与上一个窗口解绑

imageButton.Bind(window);

现在就正常绘制了
并且对点击正确做出了反应

色块按钮

d2dColorButton
以纯色为背景的按钮控件

与图片按钮类似的初始化方式

colorButton.Init(
    500, 200,//设置显示位置
    150, 50,//设置显示宽度
    L"色块按钮", D2D1::ColorF(1, 0, 1),//设置提示字符及其颜色
    D2D1::ColorF(0, 1, 0),//设置背景颜色
    ButtonCallback,//点击回调
    window,//用来提供d2d设备的窗口
    0		//用户自定义数据
);
//与窗口绑定
colorButton.Bind(window);

按钮的更多功能

可以使用[SetFontSize]函数设置字体大小

colorButton.SetFontSize(40)

使用[GetTextElement]就可以获取显示文本的窗口元素的引用，利用其进行更加详细的设置

函数控件

d2dFunctionWind
用于绘制函数视图

使用Init函数初始化

functionWindow.Init(D2D1::RectF(0, 0, windowSize.cx, windowSize.cy), window->GetD2DTargetP());

设置坐标轴格式，有两种方法

数字风格

functionWindow.SetCoordinate(
    1, 10, 1,//设置x轴最小值和最大值及其精度
    -5, 5, 1,//设置y轴最小值和最大值及其精度
    window->GetD2DTargetP()//d2d设备
);

字符串风格

functionWindow.SetCoordinate(
    0, 10,//x轴的最小值和最大值，这里代表设置函数时的刻度尺
    0, 10,//y轴的最小值和最大值，这里代表设置函数时的刻度尺
    { L"张三",L"李四" ,L"王五" ,L"伯" ,L"仲" },//每个标号的字符串，标号个数就是分割的个数
    { L"老师",L"运动员" ,L"医生" ,L"商人" ,L"农夫" },
    window->GetD2DTargetP()//d2d设备
);

设置好后便可以添加要显示的函数
使用[AddFunction]函数添加

functionWindow.AddFunction(
    xList, yList, //表示x位置的数组和表示y轴位置的数组
    1, //函数线条id
    D2D1::ColorF(1, 0, 0), //函数线条的颜色
    window->GetD2DTargetP()//d2d设备
);

比如这里使用数字形式的坐标轴绘制sin函数

std::vector<float>xList(50);
std::vector<float>yList(50);
for (int i = 0; i < 50; i++)
{
    float x = 10 / 50.f * i;
    xList[i] = x;
    yList[i] = std::sin(x) * 2;
}
functionWindow.AddFunction(
    xList, yList, //表示x位置的数组和表示y轴位置的数组
    1, //函数线条id
    D2D1::ColorF(1, 0, 0), //函数线条的颜色
    window->GetD2DTargetP()//d2d设备
);

编辑框控件

d2dEdit控件用于接受用户的自定义输入
由于需要捕获按键输入，所以当Edit进入焦点时会截断自定义的字符回调，并在结束回调后恢复 支持设置仅输入数字，多行单行切换，鼠标点击与按键移动光标，以及设置回调

编辑框控件仍使用Init函数初始化

edit.Init(
    window,//d2d设备
    L"请输入数字"//初始文本
);

还需要设置一下显示绘制，否则不会显示

edit.SetShowRect(D2D1::RectF(200, 200, 400, 300));

在设置绘图回调并成功显示后会发现无法点击，因为需要检测点击还需要与窗口绑定

edit.Bind(window);

现在就能正常的使用了，并支持中文输入
你会发现点击回车不会换行而是退出了焦点，默认状态是单行编辑，回车代表输入结束（点击编辑框外区域也代表输入结束）

觉得颜色太丑了？可以使用
SetCursorColor设置光标颜色
SetTextColor设置文本颜色
SetDefBKColor设置未在焦点时的背景颜色
SetSelectedBKColor设置焦点状态时的背景颜色

示例

edit.SetCursorColor(D2D1::ColorF(1, 0, 1), window);
edit.SetTextColor(D2D1::ColorF(1, 1, 0), window);
edit.SetDefBKColor(D2D1::ColorF(0, 0.6, 0), window);
edit.SetSelectedBKColor(D2D1::ColorF(0.3, 0.3, 0.3), window);

动画控件

动画控件主要用于播放图片动画

支持多图片切换的动画播放和单图片切片的动画播放
比如这里有一个瓦片地图来自https://kenney.nl/assets/pixel-platformer

动画控件存在两种

[d2dPictureAnimation]单图片动画控件
[d2dPicturesAnimation]多图片动画控件

这里使用单图片动画控件来实现1秒播放一个数字的计数器

首先需要使用窗口元素中的图片元素加载图片资源

//定义一个新的图片窗口元素作为数据载体
d2dPicture m_AnimationData;
//定义单图片动画控件
d2dPictureAnimation m_Animation;

在初始化函数中初始化 

//加载瓦片地图资源
m_AnimationData.LoadPictureFromFile(L"瓦片地图.png", window);

动画控件的初始化有两种方法可用

首先是对于整个图片都是有效数据且瓦片间没有间隔的，比如当前图片

m_Animation.Init(
    &m_AnimationData,	// 图片数据载体
    20,					// 图片中瓦片横排个数
    9,					// 图片中瓦片的竖排个数
    10,					// 索引起始点x坐标，以0开始
    8,					// 索引起始点y坐标
    10					// 有效瓦片个数
);

对照着瓦片地图，就能看见这是以最后10个数字为索引设置播放的

另外一种方式是更加精细的操作，允许自定义瓦片的间隔，但仍要求为有矩阵的规律的样式
比如这个图片的间隔间有空隙，我们就使用这种初始化方式来消除间隙

首先要知道图片的基础信息

//获取加载图片的大小，以像素为单位，由于图片来自网络，我不了解图片详细信息
auto pictureSize = m_AnimationData.GetImageSize();
//观察可以发现，图片的瓦片是20*9，每个读取步长就是长度除以瓦片数
Vector::Vec2 Shift(pictureSize.width / 20, pictureSize.height / 9);

然后初始化

m_Animation.Init(
    &m_AnimationData,// 图片数据载体
    Vector::Vec2(Shift.x*10,Shift.y*8),//像素的起始点坐标
    Vector::Vec2(Shift-1),//单个图片的像素宽度,假设边框的宽度为1
    Shift,//每个图片的偏移宽度
    10//图片数量
);

初始化完成后设置基础信息

//跳转到第0个索引对应的位置，也就是瓦片地图上对应的数字0，
//默认位置为0，但会因初始化发生改变，导致第一二帧出现跳帧的情况
//所以初始化后主动跳转更好
m_Animation.ToPicture(0);
//设置图片的切换时间为1秒，默认就是1秒
m_Animation.SetSwitchTime(1.0);
//设置控件的显示宽度为100
m_Animation.SetShowWide(100);
//设置控件的显示位置
m_Animation.SetShowPosition(200, 200);

在绘图回调中设置后
便可以看见一个每1秒切换一张图片的动画了
