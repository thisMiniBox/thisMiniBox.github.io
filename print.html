编辑版本0.22

简介

OneKeyD2D是一个纯头文件的开源窗口库，封装了win32 gdi（简单封装）窗口和d2d（Direct2D，之后全部简称d2d）窗口的创建与使用。

采用元素的方法分类窗口组件，模块化的编程。d2d的绘制与传统的win32 gdi相比效率更高、更稳定

相比其它窗口库，OneKeyD2D有了场景坐标、摄像机坐标、窗口坐标的概念，实现各页面的随心切换与ui的自动管理，并提供瓦片地图、柏林噪声工具，帮助初学者快速制作简单的游戏。

配置项目

由于OneKeyD2D使用的是utf-8编码保存，可能会对初学者使用造成困惑，这里提供几种编辑器的项目配置教程。

Visual Studio

这里使用Visual Studio 2022 Preview演示

将OneKeyD2D的代码复制到项目所在文件夹

然后创建一个c++文件
设置属性->C/C++->命令行
添加命令/utf-8

添加OneKeyD2D的头文件引用

最后设置你的所有项目文件的保存格式为utf-8
如果出现错误C2001常量中有换行符时，就是你的文件保存格式不对，在高级文件保存中设置保存格式为Unicode(UTF - 8无签名) - 代码页65001

Visual Studio Code

如果你一般使用的时此编译器编程
相信你能正确设置引用

主要是调试环境的配置
其中，需要包含库

"-ld2d1",
"-ldwrite",
"-lole32",
"-lwindowscodecs",
"-Winmm"，

将这些放入tasks.json文件的args部分

"args": [
    "-fdiagnostics-color=always",    //以彩色字符输出警告（非必要）
    "-g",    //生成调试信息
    "${file}",    //vs自动添加文件
    "-o",    //输出
    "${fileDirname}\\${fileBasenameNoExtension}.exe",    //当前项目地址

    //以下是库必要的链接库
    "-ld2d1",
    "-ldwrite",
    "-lole32",
    "-lwindowscodecs",
    "-lWinmm",
    //结束
    "-finput-charset=UTF-8",    //指定输入的文件格式
],

注意，使用g++编码仍会在设置窗口标题时出现乱码问题，其他部分字符能运行正常，所有一般推荐用vs(主要是因为他不会乱码)

创建窗口

之后都使用vs做示例

创建单个空窗口

首先引用头文件

#include<OneKeyD2D/OneKeyD2D.hpp>

这个头文件会以正确的顺序引用所有头文件，推荐使用这个，安全并且方便
如果比较介意全部引用，可以使用

#include<OneKeyD2D/MainWind.hpp>

这样只会引用基础库 MainWind.hpp

使用命名空间Game，所有的组件都在此命名空间中

using namespace Game;

现在可以直接创建一个d2d窗口

int main()
{
	//窗口索引标识
	MainWind_D2D window;
	//创建窗口
	window.CreateWind();
	//开启消息循环
	window.StaticWindowMessageLoop();
	return 0;
}

运行后就是一个什么都没有的空窗口

现在试试多窗口及嵌套窗口

先使用AutoClearWindBackground函数来实现窗口背景的切换和区分不同的窗口
先设置之前创建的window背景为绿色（要在消息循环前调用）

window.AutoClearWindBackground(RGB(0,255,0));

再按照之前的方法创建窗口2并设置为黄色

//窗口索引标识
MainWind_D2D window,window2;
//创建窗口
window.CreateWind();
window2.CreateWind();

window.AutoClearWindBackground(RGB(0,255,0));
window2.AutoClearWindBackground(RGB(255, 255, 0));

//开启消息循环
window.StaticWindowMessageLoop();

运行后成功创建两个不同的窗口

现在试试父子窗口
因为子窗口会遮挡父窗口的画面，所有父窗口一般使用gdi窗口便足够了
使用MainWind_GDI创建gdi窗口

MainWind_GDI fWindow;
//创建窗口，需要在子窗口创建之前创建
fWindow.CreateWind();

然后将窗口句柄作为参数放入CreateWind函数

window.CreateWind(fWindow.GethWnd());
window2.CreateWind(fWindow.GethWnd());

使用GethWnd函数便可以获取以创建成功的窗口的窗口句柄
现在运行 为什么只有一种颜色，而且移动窗口时会闪烁？
因为当创建为子窗口时，会默认占用所有客户区，现在两个子窗口的显示区域完全重叠了，所以我们应该使用MoveWind函数来移动窗口位置，让他们各占用一半

//获取父窗口的大小
auto fsize = fWindow.GetWindSize();
//移动子窗口让他们分别到左右两边
window.MoveWind(0, 0, fsize.cx * 0.5, fsize.cy);
window2.MoveWind(fsize.cx * 0.5, 0, fsize.cx, fsize.cy);

MainWind::GetWindSize()

现在的运行效果就对了

完整代码

#include<OneKeyD2D/OneKeyD2D.hpp>

using namespace Game;
int main()
{
	//窗口索引标识
	MainWind_D2D window,window2;
	MainWind_GDI fWindow;
	//创建窗口
	fWindow.CreateWind();
	window.CreateWind(fWindow.GethWnd());
	window2.CreateWind(fWindow.GethWnd());

	window.AutoClearWindBackground(RGB(0,255,0));
	window2.AutoClearWindBackground(RGB(255, 255, 0));

	//获取父窗口的大小
	auto fsize = fWindow.GetWindSize();
	//移动子窗口让他们分别到左右两边
	window.MoveWind(0, 0, fsize.cx * 0.5, fsize.cy);
	window2.MoveWind(fsize.cx * 0.5, 0, fsize.cx, fsize.cy);

	//开启消息循环
	window.StaticWindowMessageLoop();
	return 0;
}
