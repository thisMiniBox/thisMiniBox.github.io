简介

OneKeyD2D是一个纯头文件的开源窗口库，封装了win32 gdi（简单封装）窗口和d2d（Direct2D，之后全部简称d2d）窗口的创建与使用。

采用元素的方法分类窗口组件，模块化的编程。d2d的绘制与传统的win32 gdi相比效率更高、更稳定

相比其它窗口库，OneKeyD2D有了场景坐标、摄像机坐标、窗口坐标的概念，实现各页面的随心切换与ui的自动管理，并提供瓦片地图、柏林噪声工具，帮助初学者快速制作简单的游戏。

创建单个空窗口

首先引用头文件

#include<OneKeyD2D/OneKeyD2D.hpp>

这个头文件会以正确的顺序引用所有头文件，推荐使用这个，安全并且方便
如果比较介意全部引用，可以使用

#include<OneKeyD2D/MainWind.hpp>

这样只会引用基础库 MainWind.hpp

使用命名空间Game，所有的组件都在此命名空间中

using namespace Game;

现在可以直接创建一个d2d窗口

int main()
{
	//窗口索引标识
	MainWind_D2D window;
	//创建窗口
	window.CreateWind();
	//开启消息循环
	window.StaticWindowMessageLoop();
	return 0;
}

如果你实际上手操作后可能会发现背景图片对不齐
由于加载位图后默认会把绘制区域设置为图片的大小，要让图片的大小与窗口对应就需要设置窗口大小回调，让图片的显示大小跟随窗口做出调整

首先更具d2d窗口窗口大小调整回调的函数定义要求定义函数

void WindowSize(MainWind_D2D* window,int w,int h)
{
	WindowElements* windowElements = (WindowElements*)window->GetUserData();
	windowElements->background.SetShowWide(w, h);
}

然后一样的，再与窗口绑定

这样理论上就可以就能正确的显示背景了
但是我们发现并没有正确显示，只有调整窗口大小后才对。这是由于在创建窗口时没有触发窗口大小调整回调，所有我们在初始化时就设置图片的大小就行了

我们在加载图片函数后面加上这个函数

auto windowSize = window->GetWindSize();
background.SetShowWide(windowSize.cx, windowSize.cy);

这样就正常显示了
